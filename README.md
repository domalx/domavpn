# 代理程序 - 内网穿透解决方案

## 项目简介

本项目是一个基于 Python Flask 框架实现的轻量级代理程序，用于实现内网服务的外网访问。通过三层架构设计，实现了内网HTTP服务、远程代理服务器的完整链路。

## 功能特点

- **Flask框架**：使用Flask框架构建HTTP服务，稳定可靠
- **自动连接**：内网服务启动后自动连接远程代理服务器
- **随机端口分配**：远程服务器自动分配随机端口给内网服务
- **安全认证**：支持用户名密码认证，密码使用SHA256加密存储
- **IP黑名单**：连续3次认证失败的IP将被拉黑5分钟
- **HTTP访问**：用户通过HTTP方式访问代理地址和端口
- **JSON配置**：支持通过配置文件灵活设置端口地址信息
- **跨平台**：纯Python实现，支持Windows、Linux、macOS

## 实现原理

### 架构设计

```
┌─────────────────────────────────────────────────────────────────────┐
│                         客户端（用户）                               │
│              访问 http://<server-ip>:<random-port>/[path]            │
└───────────────────────────┬─────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────────┐
│                    远程代理服务器（Remote Server）                    │
│              监听端口: 8871                                         │
│              API认证 → 分配随机端口 → 转发请求到内网服务              │
└───────────────────────────┬─────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────────┐
│                     内网HTTP服务（Local Server）                     │
│              监听端口: 5000                                         │
│              启动后自动认证 → 连接代理端口 → 提供文件服务             │
└─────────────────────────────────────────────────────────────────────┘
```

### 工作流程

1. **启动远程服务器**：监听端口8871，等待内网服务连接
2. **启动内网服务**：启动HTTP服务，自动向远程服务器认证
3. **认证成功**：远程服务器分配随机端口（50000-60000）
4. **建立连接**：内网服务连接到分配的代理端口
5. **用户访问**：用户通过 `http://<server-ip>:<random-port>` 访问内网服务

### 安全特性

#### 1. 密码加密存储

远程服务器配置文件中存储密码的SHA256哈希值：

```python
password_hash = hashlib.sha256(password.encode()).hexdigest()
```

#### 2. IP黑名单机制

- 连续3次认证失败自动拉黑
- 黑名单持续5分钟
- 支持手动移除黑名单IP

## 配置说明

### 远程服务器配置 (remote_server/config.json)

```json
{
    "listen_host": "0.0.0.0",
    "listen_port": 8871,
    "auth": {
        "enabled": true,
        "username": "admin",
        "password": "password123"
    }
}
```

### 内网服务配置 (local_server/config.json)

```json
{
    "host": "127.0.0.1",
    "port": 5000,
    "remote_server": {
        "host": "127.0.0.1",
        "port": 8871,
        "auth": {
            "username": "admin",
            "password": "password123"
        }
    }
}
```

### 配置说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| listen_host | 远程服务器监听地址 | 0.0.0.0 |
| listen_port | 远程服务器监听端口 | 8871 |
| auth.enabled | 是否启用认证 | true |
| auth.username | 用户名 | admin |
| auth.password | 密码 | password123 |
| host (local) | 内网服务监听地址 | 127.0.0.1 |
| port (local) | 内网服务端口 | 5000 |
| remote_server.host | 远程服务器地址 | 127.0.0.1 |
| remote_server.port | 远程服务器端口 | 8871 |

## 服务角色说明

### 角色1：内网HTTP服务

- **运行位置**：内网主机（如用户本地电脑）
- **功能**：提供文件访问服务，启动后自动连接远程服务器
- **端口**：5000（可配置）

### 角色2：远程代理服务器

- **运行位置**：公网云服务器
- **功能**：接收内网服务认证，分配随机端口，转发用户请求
- **端口**：8871（可配置），随机分配端口从50000开始

### 角色3：客户端

- **运行位置**：用户设备
- **功能**：通过远程服务器的随机端口访问内网服务
- **访问地址**：`http://<server-ip>:<random-port>`

## 部署注意事项

### 云服务器配置

1. **开放端口**：在云服务器安全组中开放8871端口和50000-60000端口范围
2. **公网IP**：确保服务器有公网IP地址
3. **防火墙**：关闭或配置防火墙允许端口访问

### 内网服务配置

1. **配置remote_server.host**：设置为远程服务器的公网IP地址
2. **网络可达性**：确保内网主机可以访问远程服务器

### 安全建议

1. **端口限制**：限制远程服务器端口只允许特定IP访问
2. **HTTPS**：生产环境建议使用HTTPS加密传输
3. **强密码**：使用复杂密码，定期更换
4. **监控日志**：定期检查日志，关注异常登录尝试

## 常见问题

### Q1: 内网服务无法连接远程服务器

**可能原因**：
- 远程服务器未启动
- 云服务器安全组未开放端口
- 网络防火墙阻止连接

**解决方法**：
- 检查远程服务器状态
- 在云服务器安全组中开放8871端口
- 检查防火墙配置

### Q2: 认证失败

**可能原因**：
- 用户名或密码错误
- IP已被拉黑

**解决方法**：
- 检查配置文件中的用户名密码
- 检查是否被拉黑：`curl http://<server-ip>:8871/api/blacklist`
- 移除黑名单：`curl -X DELETE http://<server-ip>:8871/api/blacklist/<ip>`

### Q3: 无法通过代理端口访问

**可能原因**：
- 内网服务断开连接
- 端口范围未开放
- 端口已过期

**解决方法**：
- 检查内网服务状态
- 在云服务器安全组中开放50000-60000端口范围
- 重启内网服务重新获取端口

### Q4: 响应时间过长

**可能原因**：
- 网络延迟较高
- 服务负载过大

**解决方法**：
- 检查网络连接质量
- 考虑使用异步IO提高并发性能
